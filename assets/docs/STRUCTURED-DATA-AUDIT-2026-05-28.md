### Structural Issues Found and Fixed

#### Issue 1 â€” BreadcrumbList nested inside SoftwareApplication (high impact)

**Before:**
```json
{
  "@graph": [
    {
      "@type": "SoftwareApplication",
      "breadcrumb": {
        "@type": "BreadcrumbList",
        "itemListElement": [...]
      }
    }
  ]
}
```

**Problem:** Nesting the BreadcrumbList prevents Google's Rich Results parser from
evaluating it as an independent Breadcrumb rich result. A nested entity cannot appear
as a top-level rich result type in the SERP.

**After (fixed in `scripts/inject-jsonld.py`):**
```json
{
  "@graph": [
    {
      "@type": "SoftwareApplication",
      "breadcrumb": { "@id": "https://glee-fully.tools/toolbox/.../#breadcrumb" }
    },
    {
      "@type": "BreadcrumbList",
      "@id": "https://glee-fully.tools/toolbox/.../#breadcrumb",
      "itemListElement": [...]
    }
  ]
}
```

BreadcrumbList is now a first-class `@graph` node with its own `@id`.
SoftwareApplication references it by `@id` â€” valid linked-data pattern.
This enables Google to parse **both** a SoftwareApplication rich result **and** a
Breadcrumb rich result from the same page.

**Scope:** All 42 tool-ette pages, 8 branch-hub pages, 8 supporting pages â€” 57 pages total,
regenerated by re-running `scripts/inject-jsonld.py`.
