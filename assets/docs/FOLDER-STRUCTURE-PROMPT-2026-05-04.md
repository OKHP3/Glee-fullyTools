## Background

All three sites share the same philosophy: pure HTML/CSS/JS, no build system, GitHub Pages compatible, published by the human owner directly — the agent never pushes or commits. The folder structure described here keeps crawlable page files at the root (and in page-slug sub-directories), while everything that supports development — scripts, templates, audit data, documentation — lives in `assets/` or `scripts/`, both of which are excluded from all HTML-walking validators and indexers by a shared SKIP_DIRS rule.

The Glee-fully Tools project has established this structure to the highest standard. The other two projects should adopt it exactly, substituting their own brand names, content types, and file counts where noted.

## Folder-by-Folder Purpose & Rules

### `assets/` (root)

**What it is:** The non-crawlable project-support container. Nothing inside `assets/` is a real page that should ever be served to a site visitor or indexed by a search engine.

**Rule:** Every Python script that walks HTML files must skip `assets/` (and `node_modules/`, `.local/`, `.git/`, `attached_assets/`). This is the SKIP_DIRS constant:
```python
SKIP_DIRS = {"node_modules", ".local", ".git", "attached_assets", "assets"}
```

### `assets/audit/`

**What it is:** Machine-readable JSON output written by the three validator scripts each time they run. Files are date-stamped so historical runs are preserved alongside new ones.

**Files in Glee-fully Tools:**
| File | Written by | Purpose |
|---|---|---|
| `validation-report-2026-05-03.json` | `scripts/validate-site.py` | Per-page metadata pass/fail, issue counts |
| `links-report-2026-05-03.json` | `scripts/check-links.py` | Internal link inventory + sitemap parity |
| `asset-inventory-2026-05-03.json` | `scripts/audit-assets.py` | Every image: referenced/orphaned, pages list |

**Naming convention:** `[report-type]-[YYYY-MM-DD].json`

**What to create in OverKill Hill / AskJamie:**
- Same three files, same naming convention, same date stamp.
- Run the three validator scripts at least once to generate the initial JSON before writing any audit Markdown.
- Do NOT hand-write these files — they must always be generated by the scripts.

### `assets/css/`

**What it is:** A single stylesheet. All three sites share a common `theme.css` that is scoped by brand using body-class selectors.

**File in Glee-fully Tools:** `theme.css` (4,265 lines)

**Scope map inside `theme.css`:**
| Scope | Body class | What lives here |
|---|---|---|
| GLOBAL | (unscoped) | Tokens, reset, layout, header/nav, footer, search |
| OVERKILL | `body:not(.glee-main):not(.askjamie-main)` | OverKill Hill chrome |

| ASKJAMIE | `.askjamie-main` | AskJamie brand chrome |
| CROSS-BRAND | `.glee-main, .askjamie-main` | Shared Glee + AskJamie overrides |

Each scope is separated by a `╔══╗` boxed banner comment so edits stay in the right scope.

**What to create in OverKill Hill / AskJamie:**
- Keep `assets/css/theme.css` as the single file name — do not split, do not rename.
- When syncing CSS changes between projects: differences inside GLOBAL and CROSS-BRAND blocks should be reconciled across all three repos; differences inside OVERKILL/GLEE/ASKJAMIE blocks are intentional brand divergence.

### `scripts/`

**What it is:** Python automation — validators, content injectors, index/asset/feed generators. These run in the Replit terminal only; they are never shipped to GitHub Pages.

**Scripts in Glee-fully Tools:**

*Mutators (idempotent, AUTOGEN-marker-driven — safe to re-run):*
| Script | What it does |
|---|---|
| `scripts/normalize-head.py` | Canonical favicon chain + theme-color meta on all pages |
| `scripts/activate-icons.py` | Swaps hero `<img>` and `og:image` to per-tool GPT icons |
| `scripts/inject-jsonld.py` | Writes JSON-LD `@graph` + `BreadcrumbList` inside `<!-- AUTOGEN:JSONLD -->` markers |
| `scripts/inject-breadcrumb.py` | Writes visible `<nav aria-label="Breadcrumb">` inside `<!-- AUTOGEN:BREADCRUMB -->` markers |

*Index / asset / feed generators:*
| Script | What it does |
|---|---|
| `scripts/build-search-index.py` | Walks every `*.html`, writes `assets/data/search-index.json` |
| `scripts/audit-assets.py` | Walks `assets/img/`, writes `assets/data/icon-map.json` + `assets/audit/asset-inventory-*.json` |
| `scripts/generate-feed.py` | Generates `/feed.xml` from `search-index.json` |

*Validators (exit non-zero on regressions — wire into CI or run before every commit):*
| Script | What it checks |
|---|---|
| `scripts/validate-site.py` | Every page: title, h1, description, canonical, og:url, robots, JSON-LD, theme-color, favicon, script wiring. Writes `assets/audit/validation-report-*.json`. |
| `scripts/check-links.py` | Every internal `href` validated against filesystem + sitemap parity. Writes `assets/audit/links-report-*.json`. |

*Superseded (do not re-run):*
| Script | Status |
|---|---|
| `scripts/generate-templates.py` | Superseded by the flat structural template library. Retained for reference only. |

**Run order matters:**
```bash
# 1. Mutators first (inject-jsonld reads og:image set by activate-icons;
#    inject-breadcrumb reads JSON-LD written by inject-jsonld)
python3 scripts/normalize-head.py
python3 scripts/activate-icons.py
python3 scripts/inject-jsonld.py
python3 scripts/inject-breadcrumb.py

# 2. Index / asset / feed generators
python3 scripts/build-search-index.py
python3 scripts/audit-assets.py
python3 scripts/generate-feed.py

# 3. Validators last (read the final state of the site)
python3 scripts/validate-site.py   # must exit 0
python3 scripts/check-links.py     # must exit 0
```

**What to create in OverKill Hill / AskJamie:**
- Start with the three validators (`validate-site.py`, `check-links.py`, `audit-assets.py`) and the search-index builder (`build-search-index.py`). These are site-agnostic and should be ported directly, changing only brand strings and SKIP_DIRS.
- Add `inject-jsonld.py` and `inject-breadcrumb.py` once JSON-LD and breadcrumbs are structured the same way.
- Add `generate-feed.py` if the site has an Atom feed.
- Add `normalize-head.py` and `activate-icons.py` only if those normalization needs exist.
- All scripts must write machine-readable JSON to `assets/audit/` and human-readable output to stdout.

## Universal Rules (apply to all three projects)

1. **Never crawlable:** `assets/` and `scripts/` must never contain HTML pages. No `index.html` inside either.
2. **SKIP_DIRS:** Every Python script that walks HTML must skip `{"node_modules", ".local", ".git", "attached_assets", "assets"}`.
3. **Single stylesheet:** One `theme.css` only. Brand scoping happens through body-class selectors, not separate files.
4. **Single JS bundle:** One `app.js` only per site. No per-feature scripts loaded on every page.
5. **Root-absolute paths:** All `<link>`, `<script>`, `<img src>`, and `<a href>` to internal resources use `/assets/...` paths (not `../` relative paths). This works correctly regardless of URL depth.
6. **Validators exit 0:** `validate-site.py` and `check-links.py` must exit 0 before any content is pushed to GitHub. Wire them as the final step in every session.
7. **Templates are structural, not per-page:** One template per distinct page type. Token syntax: `[[UPPERCASE-KEBAB-CASE]]`. Each template has a complete token list in a comment block before `<!DOCTYPE html>`.
8. **Audit docs are dated:** Every `AUDIT_*` and `FINAL_AUDIT_*` file in `assets/docs/` carries the date it was produced (`YYYY-MM-DD`). Regenerate and re-date on every significant audit pass.
9. **`gleefully-replit-theme.json` / `[brand]-replit-theme.json`:** This file is owned by the Replit app theme system. Never edit, move, or delete it.
10. **GitHub Pages favicons:** `/favicon.ico` must exist at the repo root. The full favicon family lives in `assets/img/favicons/` and is referenced in every `<head>` with root-absolute paths.
