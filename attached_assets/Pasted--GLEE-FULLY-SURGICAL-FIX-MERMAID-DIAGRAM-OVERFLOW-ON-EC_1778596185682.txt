# GLEE-FULLY — SURGICAL FIX: MERMAID DIAGRAM OVERFLOW ON ECOSYSTEM PAGE
**File:** ecosystem/index.html (primary) + assets/css/ (shared fix)
**Live URL:** https://glee-fully.tools/ecosystem/
**Symptom:** Mini Mermaid branch diagrams overflow their parent card containers —
specifically visible on #02 Treasured Finds (7 Tool-ettes) and any branch
with more nodes than its neighbors.

---

## ROOT CAUSE

Mermaid.js renders its diagrams by injecting an `<svg>` element with **explicit
inline `width` and `height` attributes** derived from the node count at runtime.
Example of what Mermaid emits:

```html
<svg width="842" height="312" ...>
```

Because the `width` attribute is inline on the element itself (not a CSS property),
it overrides any CSS `max-width` rule on the SVG. The card containers do not have
`overflow: hidden`, so the oversized SVG bleeds out of the card boundary and
overlaps adjacent cards.

The #02 Treasured Finds diagram has 7 Tool-ette nodes (the most on the page).
#01 has 6, #03 has 5, #04 has 5 — so #02 renders wider/taller and is the most
visibly broken, but this is a systemic issue that will affect any branch with
more nodes than its neighbors at any viewport width.

## STEP 2 — CSS FIX (apply first)

### 2.1 Find the primary stylesheet

```bash
# Identify the main CSS file
grep 'rel="stylesheet"' ecosystem/index.html | grep assets
```

Open the primary CSS file (likely `assets/css/main.css` or `assets/css/style.css`).

### 2.2 Add the containment rules

Append the following CSS block to the primary shared stylesheet. Do NOT add it as
an inline `<style>` block in the HTML — it goes in the external file.

```css
/* ============================================================
   MERMAID DIAGRAM CONTAINMENT
   Fixes: branch diagrams overflowing card boundaries on /ecosystem/
   Root cause: Mermaid injects inline width/height on <svg>;
   overflow:hidden on the wrapper and attribute-level override
   via JS (see assets/js/mermaid-contain.js) are both required.
   ============================================================ */

/* Card-level containment — prevent ANY child from bleeding out */
.branch-card,
.tool-branch-card,
.ecosystem-card,
[class*="branch"][class*="card"] {
  overflow: hidden;
  position: relative;
}

/* Mermaid wrapper containment */
.mermaid,
.mermaid-diagram,
[class*="mermaid"] {
  width: 100%;
  max-width: 100%;
  overflow: hidden;
  display: block;
}

/* Force the SVG itself to respect its container width.
   The !important is required because Mermaid sets width inline. */
.mermaid svg,
.mermaid-diagram svg {
  width: 100% !important;
  max-width: 100% !important;
  height: auto !important;
  display: block;
}
```

**Note on class names:** The selectors above use common naming patterns.
Before writing, confirm the actual class names on the card containers
by inspecting `ecosystem/index.html`. Replace `branch-card`,
`tool-branch-card`, and `ecosystem-card` with whatever classes are
actually used. The `[class*="branch"][class*="card"]` attribute selector
is a fallback that catches compound class names regardless of order.

### 2.3 If no shared class exists on the card wrappers

## STEP 3 — JAVASCRIPT POST-RENDER FIX (required — CSS alone is insufficient)

CSS `width: 100% !important` on the SVG element will work in most browsers, but
Mermaid sometimes sets `width` as an **HTML attribute** (`<svg width="842">`) in
addition to an inline style. HTML attributes have lower specificity than CSS for
some properties, but the interaction is browser-dependent. The reliable fix is to
strip the inline `width` and `height` attributes after Mermaid renders, then let
CSS control sizing.

### 3.1 Create assets/js/mermaid-contain.js

```javascript
/**
 * mermaid-contain.js
 * Glee-fully Personalizable Tools™
 *
 * Post-render hook: strips inline width/height from all Mermaid SVGs
 * so CSS containment can control their sizing within card boundaries.
 *
 * Must load AFTER mermaid.js has finished rendering.
 * Uses MutationObserver to catch async renders, then disconnects.
 */

(function () {
  'use strict';

  /**
   * Strip inline width/height attributes from a Mermaid SVG element.
   * Leaves viewBox intact — that's what preserves the diagram's
   * internal proportions when CSS controls the outer size.
   */
  function containMermaidSvg(svg) {
    // Remove the inline attributes Mermaid injects
    svg.removeAttribute('width');
    svg.removeAttribute('height');
    svg.removeAttribute('style'); // Mermaid may also set max-width inline

    // Ensure the SVG has a viewBox (Mermaid should have set this already)
    // If viewBox is missing, the diagram will collapse — don't remove width in that case
    if (!svg.getAttribute('viewBox')) {
      // Re-derive viewBox from the current rendered dimensions
      var bbox = svg.getBoundingClientRect();
      if (bbox.width > 0 && bbox.height > 0) {
        svg.setAttribute('viewBox', '0 0 ' + Math.round(bbox.width) + ' ' + Math.round(bbox.height));
      }
      // If we can't get dimensions, restore width to avoid collapse
      else {
        return; // bail — better to overflow than to disappear
      }
    }
  }

  /**
   * Process all currently-rendered Mermaid SVGs on the page.
   */
  function containAll() {
    var svgs = document.querySelectorAll('.mermaid svg, .mermaid-diagram svg');
    svgs.forEach(function (svg) {
      containMermaidSvg(svg);
    });
    return svgs.length;
  }

  /**
   * Mermaid renders asynchronously. We need to wait for it to finish.
   * Strategy:
   * 1. Try immediately (in case Mermaid already ran synchronously)
   * 2. Watch for SVG elements being added to .mermaid containers
   * 3. Disconnect once all expected diagrams are rendered
   */
  function init() {
    // Attempt immediate pass
    var count = containAll();

    // Count how many .mermaid containers exist on this page
    var containers = document.querySelectorAll('.mermaid, .mermaid-diagram');
    var expected = containers.length;

    if (count >= expected && expected > 0) {
      // All rendered synchronously — done
      return;
    }

    // Set up MutationObserver for async renders
    var observer = new MutationObserver(function (mutations) {
      var triggered = false;
      mutations.forEach(function (mutation) {
        mutation.addedNodes.forEach(function (node) {
          if (node.nodeName === 'svg') {
            containMermaidSvg(node);
            triggered = true;
          }
        });
      });

      // Check if all diagrams are now rendered; if so, disconnect
      if (triggered) {
        var rendered = document.querySelectorAll('.mermaid svg, .mermaid-diagram svg').length;
        if (rendered >= expected) {
          observer.disconnect();
        }
      }
    });

    // Observe all Mermaid containers for child additions
    containers.forEach(function (container) {
      observer.observe(container, { childList: true, subtree: true });
    });

    // Safety timeout: disconnect observer after 10 seconds regardless
    setTimeout(function () {
      observer.disconnect();
      // Final cleanup pass
      containAll();
    }, 10000);
  }

  // Run after DOM is ready
  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', init);
  } else {
    // DOM already parsed — Mermaid may or may not have run yet
    // Use requestAnimationFrame to give Mermaid one render cycle first
    requestAnimationFrame(function () {
      setTimeout(init, 0);
    });
  }

})();
```

### 3.2 Load mermaid-contain.js in ecosystem/index.html

Find where mermaid.js is loaded in `ecosystem/index.html`. Add `mermaid-contain.js`
**immediately after** the Mermaid initialization script (not before — it must run
after Mermaid has been called):

```html
<!-- Load AFTER mermaid init -->
<script src="/assets/js/mermaid-contain.js" defer></script>
```

If Mermaid is initialized via an inline `<script>` block, the load order is:

```html
<script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>
<script>
  mermaid.initialize({ startOnLoad: true, ... });
</script>
<script src="/assets/js/mermaid-contain.js" defer></script>
```

The `defer` attribute ensures it runs after HTML parsing but before `DOMContentLoaded`,
which is the right window for catching Mermaid's async render.

## STEP 5 — DECIDE: GLOBAL OR ECOSYSTEM-ONLY

The Mermaid containment CSS should go in the shared stylesheet because:
- Other pages may add Mermaid diagrams in the future
- The CSS rules are harmless when no Mermaid diagrams are present
- Keeping containment in the shared file means new pages get it for free

The `mermaid-contain.js` script should be loaded **only on pages that use Mermaid**
(currently: ecosystem/index.html, and any Tool branch/Tool-ette pages that have
diagrams). Do not load it globally — it's a no-op on pages without `.mermaid`
elements, but it adds an unnecessary observer that wastes a tiny amount of memory.

```bash
# Find all pages that include mermaid
grep -rl 'mermaid' --include="*.html" . \
  | grep -v 'assets/' | grep -v 'attached_assets/' | sort
```

Add the `mermaid-contain.js` script tag to every page that appears in that list.

## CONSTRAINTS

1. Do not modify the Mermaid diagram source code (the `graph LR` blocks in the HTML).
   The fix is purely CSS + JS containment — not diagram restructuring.
2. Do not downgrade the Mermaid CDN version to "fix" this. Version is not the issue.
3. Do not set a fixed `height` on the cards. Fixed height breaks cards with more
   content at narrow viewports. Use `overflow: hidden` + dynamic sizing only.
4. Do not set `transform: scale()` on the diagrams. Scaling causes the clickable
   node hit areas to misalign from their visual position.
5. The `viewBox` attribute on the SVG must be preserved. That's what maintains
   proportions when CSS controls the outer dimensions.
6. CSS goes in `assets/css/`. JS goes in `assets/js/`. Nothing inline.
