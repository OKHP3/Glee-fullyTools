### 1.3 Color-scheme meta — standardize to `light dark`


Every page must have:

The order `light dark` means light is the preferred/default; dark is supported. `dark light` reverses this preference. `light` alone removes dark mode support entirely. Fix all pages to `light dark`.

### 5.2 Compare nav across all pages

```bash
python3 - <<'EOF'
import re, hashlib
from pathlib import Path

def extract_nav(content):
    m = re.search(r'(<nav\b.*?</nav>)', content, re.DOTALL)
    if m:
        # Normalize whitespace for comparison
        return re.sub(r'\s+', ' ', m.group(1)).strip()
    return None

homepage_nav = extract_nav(Path('index.html').read_text(encoding='utf-8', errors='replace'))
canonical_hash = hashlib.md5(homepage_nav.encode()).hexdigest() if homepage_nav else None

deviations = []
for path in sorted(Path('.').rglob('index.html')):
    if any(x in str(path) for x in ['assets/','attached_assets/','tools/']):
        continue
    if str(path) == 'index.html':
        continue
    content = path.read_text(encoding='utf-8', errors='replace')
    nav = extract_nav(content)
    if nav is None:
        deviations.append(f"NO NAV FOUND: {path}")
    elif hashlib.md5(nav.encode()).hexdigest() != canonical_hash:
        deviations.append(f"NAV DIFFERS: {path}")

if not deviations:
    print("All navs match the homepage canonical. ✓")
else:
    for d in deviations:
        print(d)
    print(f"\n{len(deviations)} pages with nav deviations")
EOF
```

For every page where the nav differs from the homepage canonical, diff the specific differences and correct them. The nav is a site constant — it must be byte-identical (whitespace normalized) across all pages.

## AUDIT DOMAIN 6 — CSS AND JS EXTERNALIZATION

This domain enforces the rule: **no inline styles or scripts on any page except where technically required.** All CSS and JS must live in `/assets/css/` and `/assets/js/` and be loaded via external references.

### 6.1 Inventory assets/css/ and assets/js/

```bash

echo "=== JS files in /assets/js/ ==="
```

Document the canonical file list. This is the inventory you'll use to determine what should be externalized.

### 6.2 Detect inline `<style>` blocks in HTML pages

```bash
python3 - <<'EOF'
import re
from pathlib import Path

for path in sorted(Path('.').rglob('*.html')):
    if any(x in str(path) for x in ['assets/','attached_assets/','tools/']):
        continue
    content = path.read_text(encoding='utf-8', errors='replace')
    styles = re.findall(r'<style\b[^>]*>(.*?)</style>', content, re.DOTALL)
    for i, style in enumerate(styles):
        size = len(style.strip())
        if size > 20:  # Ignore tiny operational styles (e.g., display:none)
            print(f"{path}: inline <style> block #{i+1} ({size} chars)")
EOF
```

For each inline `<style>` block found:
1. Determine if the CSS is page-specific or shared
2. If shared: move to the appropriate existing file in `assets/css/` or create `assets/css/components.css` if one doesn't exist
3. If page-specific: create `assets/css/page-{pagename}.css` and reference it with `<link rel="stylesheet">`
4. Remove the inline block from the HTML

**Exception — do not externalize:** `<style>` blocks that are 20 characters or fewer (e.g., `display:none` hacks), or blocks containing critical above-the-fold CSS where render-blocking externalization would hurt LCP (document this case explicitly if it applies).

### 6.3 Detect inline `<script>` blocks in HTML pages

```bash
python3 - <<'EOF'
import re
from pathlib import Path

EXEMPTIONS = [
    'application/ld+json',   # Structured data — must remain inline
    'application/json',       # Data blocks — must remain inline
]

for path in sorted(Path('.').rglob('*.html')):
    if any(x in str(path) for x in ['assets/','attached_assets/','tools/']):
        continue
    content = path.read_text(encoding='utf-8', errors='replace')
    scripts = re.findall(r'<script\b([^>]*)>(.*?)</script>', content, re.DOTALL)
    for attrs, body in scripts:
        # Skip exemptions
        if any(ex in attrs for ex in EXEMPTIONS):
            continue
        # Skip tiny GA dataLayer init (under 50 chars — operational)
        body_stripped = body.strip()
        if len(body_stripped) < 50:
            continue
        # Skip empty scripts
        if not body_stripped:
            continue
        print(f"{path}: inline <script> ({len(body_stripped)} chars)")
        print(f"  First 100 chars: {body_stripped[:100]}")
        print()
EOF
```

For each inline `<script>` block found (excluding JSON-LD and tiny operational snippets):
1. Determine if the JS is page-specific or shared
2. If shared: move to an appropriately named file in `assets/js/`
3. If page-specific: create `assets/js/page-{pagename}.js` and load it before `</body>`
4. Replace the inline block with `<script src="/assets/js/filename.js" defer></script>`

**Exception — do not externalize:**
- `<script type="application/ld+json">` blocks — structured data must remain inline
- GA4 initialization snippet — while technically inline JS, it is a vendor-required pattern and is exempt. If it is being extracted to `assets/js/analytics.js` per Domain 3, that's fine, but do not create a broken state by removing the inline version before the external version is confirmed working.

### 6.5 Verify external CSS/JS links are consistent across all pages

Every page should load the same base set of external CSS and JS (the site's shared stylesheet, Bootstrap, Google Fonts, etc.) in the same order. Variation in this set causes visual inconsistency.

```bash
python3 - <<'EOF'
import re, hashlib
from pathlib import Path

def extract_resource_links(content):
    css = sorted(re.findall(r'<link[^>]+rel=["\']stylesheet["\'][^>]+href=["\']([^"\']+)["\']', content))
    js_src = sorted(re.findall(r'<script[^>]+src=["\']([^"\']+)["\']', content))
    return tuple(css), tuple(js_src)

homepage = Path('index.html').read_text(encoding='utf-8', errors='replace')
canonical_css, canonical_js = extract_resource_links(homepage)

print("=== Canonical CSS (from homepage) ===")
for c in canonical_css:
    print(f"  {c}")
print("\n=== Canonical JS src (from homepage) ===")
for j in canonical_js:
    print(f"  {j}")

deviations = []
for path in sorted(Path('.').rglob('index.html')):
    if any(x in str(path) for x in ['assets/','attached_assets/','tools/']):
        continue
    if str(path) == 'index.html':
        continue
    content = path.read_text(encoding='utf-8', errors='replace')
    css, js = extract_resource_links(content)
    if css != canonical_css:
        deviations.append(f"CSS DIFFERS: {path}\n  Has: {css}\n  Expected: {canonical_css}")
    if js != canonical_js:
        deviations.append(f"JS DIFFERS: {path}\n  Has: {js}\n  Expected: {canonical_js}")

if not deviations:
    print("\nAll pages load the same external resources. ✓")
else:
    print(f"\n{len(deviations)} deviations:")
    for d in deviations[:10]:
        print(d)
EOF
```

For any page loading fewer resources than the homepage canonical set, add the missing references. For any page loading extra resources not in the canonical set, either add them to the canonical (if they should be global) or move them to page-specific loading.
