## STEP 4 — CSS ADJUSTMENT FOR HERO CONTEXT

The `universe-diagram-card` CSS from the prior fix sets `margin: 2rem 0`
which is correct when it's a standalone block below the hero. Inside the
hero right column it needs slight adjustment:

Add this rule to the existing `.universe-diagram-card` block in the
shared CSS file (do not create a new block — extend the existing one):

```css
/* Inside hero right column — remove top margin, align with hero top */
.hero-diagram-col .universe-diagram-card,
.hero-visual .universe-diagram-card {
  margin-top: 0;
  height: 100%;           /* fill the hero column height */
  display: flex;
  flex-direction: column;
}

.hero-diagram-col .universe-diagram-card .mermaid,
.hero-visual .universe-diagram-card .mermaid {
  flex: 1;               /* diagram fills available height */
  min-height: 300px;     /* minimum so it doesn't collapse */
}
```

Adjust the selector to match whatever class name is on the right column
div (from Step 2 — `hero-visual`, `hero-diagram-col`, or the Bootstrap
`col-lg-6` class if no custom class exists).
