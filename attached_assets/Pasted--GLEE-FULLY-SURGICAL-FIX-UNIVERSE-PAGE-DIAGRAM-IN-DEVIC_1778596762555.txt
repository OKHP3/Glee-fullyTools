# GLEE-FULLY — SURGICAL FIX: UNIVERSE PAGE DIAGRAM IN DEVICE MOCKUP FRAME
**File:** universe/index.html
**Live URL:** https://glee-fully.tools/universe/
**Reference (correct):** https://askjamie.bot/universe/
**Symptom:** The OKHP³™ universe diagram is trapped inside a decorative
phone/device mockup frame — far too small for a 40+ node diagram. The
Mermaid attribution callout is floating disconnected outside the frame.
The AskJamie version of the same diagram (identical source) renders
correctly in a full-width card.

---

## ROOT CAUSE — READ THIS BEFORE TOUCHING ANYTHING

The two pages share the exact same Mermaid `graph LR` source code.
The visual difference is 100% structural HTML, not diagram syntax.

**AskJamie (`askjamie.bot/universe/`):**
The diagram sits in a plain white card container at full column width.
No decorative wrapper. The Mermaid attribution is a simple `<p>` below
the diagram inside the same card.

**Glee-fully (`glee-fully.tools/universe/`):**
The diagram is wrapped in a decorative UI element — a phone or device
mockup frame (a visual design flourish that looks like a mobile device
screen, likely a `<div>` with border-radius, shadow, and constrained
dimensions meant to evoke a device). This frame is correct for simple
decorative screenshots or small previews, but completely wrong for a
dense, interactive, 40-node diagram that needs room to breathe.

Additionally, the attribution text ("This diagram is rendered with
Mermaid.AI...") is outside the frame in a floating callout box,
disconnected from the diagram it describes.

**The fix:** Remove the device mockup wrapper from around the diagram.
Replace it with the same clean card container pattern the AskJamie page
uses. Move the attribution text inside the card, below the diagram.

### 3.3 Add CSS for the universe diagram card

Add the following to the primary shared stylesheet (`assets/css/main.css`
or equivalent). Do not add it inline.

```css
/* ============================================================
   UNIVERSE PAGE — DIAGRAM CARD
   Replaces device mockup frame with a clean full-width card.
   Matches the AskJamie universe page display pattern.
   ============================================================ */

.universe-diagram-card {
  background: var(--color-paper, #f6f2ee);
  border: 1px solid rgba(42, 35, 32, 0.12);   /* espresso at 12% opacity */
  border-radius: 12px;
  padding: 2rem;
  margin: 2rem 0;
  overflow: hidden;
  width: 100%;
  box-sizing: border-box;
}

/* In dark mode */
@media (prefers-color-scheme: dark) {
  .universe-diagram-card {
    background: var(--color-dark-surface, #2a2320);
    border-color: rgba(243, 185, 50, 0.15);   /* gold at 15% opacity */
  }
}

/* The diagram itself */
.universe-diagram-card .mermaid {
  width: 100%;
  overflow-x: auto;     /* allow horizontal scroll if diagram is very wide */
  overflow-y: hidden;
}

/* Force SVG to fill the card width — same fix as mermaid-contain.js */
.universe-diagram-card .mermaid svg {
  width: 100% !important;
  max-width: 100% !important;
  height: auto !important;
  display: block;
}

/* Attribution text below the diagram */
.diagram-attribution {
  margin-top: 1.25rem;
  font-size: 0.875rem;
  color: var(--color-espresso, #2a2320);
  opacity: 0.65;
  text-align: center;
}

.diagram-attribution a {
  color: var(--color-rust, #d35b2d);
  text-decoration: none;
  font-weight: 600;
}

.diagram-attribution a:hover {
  text-decoration: underline;
}
```

**Note on overflow-x:** The universe diagram has 40+ nodes and will
naturally render wider than most screen widths. `overflow-x: auto` on
the diagram container lets users scroll horizontally to explore the full
diagram rather than forcing it to squash. This is the correct behavior
for a complex reference diagram. Do NOT force `width: 100% !important`
and `height: auto` if it causes the diagram to become illegibly small —
the horizontal scroll is preferable to an unreadable compressed chart.
