## 1 · What changed in this session

### 1.1 New utility scripts (`tools/`)
Three idempotent Python scripts now live alongside `build-search-index.py`. Re-running them is safe — they detect their own output and replace it cleanly.

| Script | Purpose |
|---|---|
| `tools/inject-jsonld.py` | Injects an autogenerated JSON-LD block (`SoftwareApplication` for tool-ettes, `CollectionPage` for branch/toolbox hubs, `WebPage` for everything else) plus a path-derived `BreadcrumbList`. Skips homepage and `/search/` (which already carry curated graphs). |
| `tools/activate-icons.py` | Maps each tool-ette / branch folder to its matching `Glee-fullyTools-GPTIcon-…-Background-RetroStripe-Square-1024.png`, rewrites `og:image`, `twitter:image`, and the `.hero-visual` `<img>` so the brand-distinct GPT art finally surfaces on the rendered site. |
| `tools/normalize-head.py` | Site-wide favicon chain normalizer: sets `theme-color` to brand rust `#d35b2d`, strips the broken `safari-pinned-tab.svg` reference, ensures every page has the modern SVG favicon, the PNG fallbacks, the ICO, the manifest link, and the `mobile-web-app-capable` meta tags. |

All three use `<!-- AUTOGEN:* -->` markers (or stable patterns) for safe re-runs. To re-process the site after content edits:

```bash
python3 tools/normalize-head.py     # 1. canonical favicon chain + theme-color
python3 tools/activate-icons.py     # 2. swap hero <img> + og:image to GPT icons
python3 tools/inject-jsonld.py      # 3. emit JSON-LD (captures the new og:image
                                    #    as primaryImageOfPage)
python3 tools/build-search-index.py # 4. rebuild assets/data/search-index.json
```
> The order matters: `inject-jsonld` reads `og:image` to set
> `primaryImageOfPage`, so it must run *after* `activate-icons`.

### 2.1 Bugs caught & fixed during code review

The architect review of this same session uncovered three issues that were then fixed in-session:
1. **`page_kind()` mis-classification** — the tool-ette regex was being applied to the *branch* folder name (`01-discovered-careers`) instead of the *tool-ette* folder name (`01a-resume-builder`). All 42 tool-ettes were initially typed as `WebPage`. Fixed: `page_kind` now requires `len(parts) == 4` and matches `^\d+[a-z]-` against `parts[2]`.
2. **JSON-LD `primaryImageOfPage` captured the old butterfly URL** because `inject-jsonld.py` had run *before* `activate-icons.py`. Fixed by re-running inject after activate; the canonical run order is now documented as `normalize → activate → inject → search-index`.
3. **`inject()` was not byte-idempotent** — its regex didn't capture the leading indent of the `<!-- AUTOGEN:JSONLD -->` line, so each re-run added 4 more spaces of indent. Fixed: regex now consumes `[ \t]*` before the START marker. Verified by running the script twice and diffing — output is now byte-identical.

Spot-checked tool-ette rendering: hero card and construction overlay still render correctly; new JSON-LD parses without warnings; brand chrome unchanged.

## 4 · How to keep this clean
The three new scripts in `tools/` are the source of truth for the auto-generated `<head>` blocks and the icon activation. The next time a tool-ette is added, simply:

1. Drop the new `Glee-fullyTools-GPTIcon-{prefix}-{Name}-Background-RetroStripe-Square-1024.png` into `assets/img/` using the existing naming convention.
2. Add the new page to `sitemap.xml`.
3. Run the four scripts in order (`normalize-head.py`, `activate-icons.py`, `inject-jsonld.py`, `build-search-index.py`). Order matters — see §1.1.

The pages that should *never* be touched by `inject-jsonld.py` (the homepage and `/search/`) are listed in the `SKIP_FILES` set at the top of that script — keep them there.
